# EasyStyle 가상 스타일리스트 - PRD (Product Requirements Document)

## 📋 프로젝트 개요

### 프로덕트명
**EasyStyle 가상 스타일리스트**

### 비전 선언문
AI 기반 개인화 스타일링 서비스로 누구나 쉽게 전문 스타일리스트의 조언을 받을 수 있는 혁신적인 패션 플랫폼을 제공한다.

### 핵심 가치 제안
- **개인화**: 사용자의 체형, 선호도, 상황에 맞춘 맞춤형 스타일링
- **편의성**: 사진 한 장과 간단한 텍스트로 완성되는 스타일링
- **즉시성**: AI를 통한 실시간 스타일 생성 및 상품 추천
- **접근성**: 전문 스타일리스트 서비스를 합리적인 비용으로 제공

## 🎯 비즈니스 목표

### 주요 목표
1. **사용자 경험 향상**: 직관적이고 간편한 스타일링 경험 제공
2. **매출 증대**: 상품 구매 연결을 통한 커미션 수익 모델 상품은 대한민국 쇼핌몰에서 구매가 가능한 상품으로 고객을 대신해서 구매대행
3. **시장 점유율 확대**: 국내 AI 패션 서비스 선도 기업 포지셔닝
4. **고객 만족도**: 90% 이상의 스타일링 만족도 달성

### 성공 지표 (KPI)
- DAU (Daily Active Users): 10,000명 (6개월 내)
- 스타일링 완료율: 85% 이상
- 구매 전환율: 15% 이상
- 사용자 만족도: 4.5/5.0 이상
- 재방문율: 70% 이상

## 👥 타겟 사용자

### 주요 타겟
1. **패션 초보자** (25-40세)
   - 스타일링에 대한 지식이 부족한 사용자
   - 전문가의 도움을 원하지만 비용 부담을 느끼는 사용자

2. **바쁜 직장인** (25-45세)
   - 시간 부족으로 쇼핑이나 스타일링에 신경 쓸 여유가 없는 사용자
   - 효율적인 패션 솔루션을 찾는 사용자

3. **트렌드 추종자** (20-35세)
   - 최신 트렌드를 반영한 스타일을 원하는 사용자
   - SNS에 올릴 만한 스타일링을 찾는 사용자

### 사용자 페르소나

**주 타겟: "바쁜 회사원 지민"**
- 나이: 28세, 직장인
- 고민: "매일 뭘 입을지 고민하는 시간이 아깝다"
- 니즈: 상황에 맞는 적절한 옷차림, 시간 절약
- 행동: 온라인 쇼핑 선호, 모바일 위주 서비스 이용

## 🔧 현재 제품 분석

### 기술 스택
- **Frontend**: React 19.1.1, TypeScript, Vite
- **AI Engine**: Google Gemini API
- **스타일링**: Tailwind CSS
- **배포**: 나노바나나를 활용한 Vercel 배포 / DB 는 supabase / admon 은  admin 이아닌 phillip 이라는 디렉토리로 

### 현재 기능 현황

#### ✅ 구현된 기능
1. **이미지 업로드**: 카메라 촬영 / 갤러리 선택
2. **AI 스타일링**: Gemini를 통한 이미지 생성 및 스타일 제안  나노바나나활용
3. **프롬프트 검증**: 사용자 요청의 구체성 검증 및 추가 질문
4. **상품 추천**: AI 기반 맞춤 상품 5개 추천
5. **상품 선택**: 개별 상품 선택/해제 기능
6. **구매 요청**: 선택 상품 또는 전체 상품 구매 요청
7. **반응형 UI**: 모바일 최적화된 인터페이스

#### 🔄 현재 화면 구성
- **Home**: 이미지 업로드 진입점
- **Styling**: 스타일 요청 입력 및 AI 질문 응답
- **Result**: 생성된 스타일 이미지 및 추천 상품 표시

### 현재 시스템 아키텍처
```
User Interface (React/TypeScript)
    ↓
Gemini Service Layer
    ├── Style Generation (gemini-2.5-flash)
    ├── Image Generation (gemini-2.5-flash-image-preview)
    ├── Product Recommendation (gemini-2.5-flash)
    └── Image Cropping (gemini-2.5-flash-image-preview)
```

## 🚀 제품 요구사항

### 핵심 기능 요구사항

#### 1. 사용자 인증 시스템 (Phase 1)
- **소셜 로그인**: Google, Kakao, Naver
- **사용자 프로필**: 기본 정보, 스타일 선호도
- **히스토리 관리**: 과거 스타일링 기록 저장/조회

#### 2. 향상된 AI 스타일링 (Phase 2)
- **다중 이미지 생성**: 3-5가지 스타일 옵션 제공
- **스타일 커스터마이징**: 생성된 스타일의 부분 수정
- **계절/날씨 고려**: 현재 날씨 정보 반영
- **체형 분석**: 사용자 체형에 맞춤 스타일링

#### 3. 고도화된 상품 연동 (Phase 3)
- **실시간 재고 확인**: 품절 상품 필터링
- **가격 비교**: 여러 쇼핑몰 가격 비교
- **할인 정보**: 쿠폰, 세일 정보 연동
- **배송 정보**: 배송비, 배송일 정보 제공

#### 4. 커뮤니티 기능 (Phase 4)
- **스타일 공유**: 생성한 스타일을 다른 사용자와 공유
- **평가 시스템**: 다른 사용자 스타일에 좋아요/댓글
- **스타일 챌린지**: 테마별 스타일링 챌린지
- **인플루언서 협업**: 패션 인플루언서와의 콜라보

### 비기능적 요구사항

#### 성능 요구사항
- **응답 시간**: 이미지 생성 30초 이내
- **동시 사용자**: 1,000명 동시 접속 지원
- **가용성**: 99.9% 업타임 보장
- **이미지 품질**: Full HD(1920x1080) 이상

#### 보안 요구사항
- **데이터 암호화**: 전송/저장 데이터 암호화
- **API 보안**: Gemini API 키 보안 관리
- **사용자 정보 보호**: GDPR 준수 개인정보 처리
- **이미지 보안**: 업로드 이미지 자동 삭제 정책

#### 호환성 요구사항
- **브라우저**: Chrome, Safari, Edge 최신 버전
- **모바일**: iOS 14+, Android 10+
- **화면 해상도**: 320px ~ 1920px 반응형 지원

   - **대응**: 다중 AI 모델 백업, 품질 검증 시스템

2. **API 의존성**: Google Gemini API 장애
   - **대응**: 다른 AI 서비스 백업 연동 (OpenAI, Anthropic)

1. **경쟁사 등장**: 유사 서비스 급증
   - **대응**: 차별화된 UX, 빠른 기능 개발, 브랜드 충성도 구축

2. **저작권 이슈**: 생성 이미지 저작권 문제
   - **대응**: 법적 검토, 이용약관 명시, 보험 가입

3. **사용자 이탈**: 초기 사용자 확보 어려움
   - **대응**: 마케팅 투자, 인플루언서 협업, 추천 프로그램

## 📈 성공 측정 방법

### 사용자 지표
- **가입률**: 월간 신규 가입자 수
- **활성도**: DAU/MAU 비율
- **리텐션**: 7일/30일 재방문율
- **세션 길이**: 평균 사용 시간

- **ARPU**: 사용자당 평균 수익
- **LTV**: 고객 생애 가치
- **CAC**: 고객 획득 비용

### 품질 지표
- **만족도**: 스타일링 만족도 설문
- **완료율**: 스타일링 프로세스 완료율
- **오류율**: 시스템 오류 발생률
- **응답시간**: API 평균 응답 시간
